::: Activity

- Provide the first interaction with user
- designed to provide visual interface
- by convenction activities should be modular in the sense activity support a single focussed thing taht use can do in app
- Aplication is compused for several activities, each with a single purpose.
- For navigations between activities, android provides several ways as:
- Tasks
- Task Backstack
- Suspeding and resuming activities


:: Tasks
- tasks is a set of related activities, these related activities don't have to be part of same application
- Most tasks start at the home screen when launches application

:: Task Backstack
- When activity is launched, it goes on top of the backstack
- When activity is destroyed, it is popped off the backstack

:: Activity Lifecycle
- Some of these actions depend on user behavior
- Some depend on Android (Kill process because memory)
